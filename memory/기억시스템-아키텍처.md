## 🏗️ 4계층 아키텍처

### Layer 1: 수집 (Ingest)
- OpenClaw 채팅 로그 → `memory/YYYY-MM-DD.md`
- 시스템 모니터링 로그
- 사용자 대화, 작업 결과 자동 기록

### Layer 2: 분석 (Synthesis) — NotebookLM
- 매일 저녁: 당일 데이터 → NotebookLM 질의
- 중복 제거, 요약, 분류
- 주간/월간 리포트 생성
- 모순점/끊어진 링크 탐지 (gbrain maintain 대체)
- **노트북 ID**: `461665a7-ed90-4307-8649-650fc6ac3e34`

### Layer 3: 영구 저장 (Wiki & Graph)
- **옵시디언 Vault**: 106개 .md 파일 (Git 버전 관리)
  - `00-코딩-프로젝트/` — VF/키 프로젝트
  - `01-일일-메모/` — 날짜별 로그
  - `02-금형정보/` — 마스터 데이터
  - `02-기술-문서/` — 가이드/시스템
- **PostgreSQL/Flask 검색** (향후 gbrain으로 대체)
- **GitHub**: https://github.com/comage9/obsidian-vault (master)

### Layer 4: 인터페이스 (Schema & Action)
- `MEMORY.md` — 프로젝트 인덱스 (장기 기억)
- `TOOLS.md` — NotebookLM 활용 규칙
- `AGENTS.md` — 에이전트 행동 규칙
- OpenClaw가 Layer 3을 먼저 조회 (Brain-first)

## 📐 근원 패턴

### llm-wiki (Andrej Karpathy)
- **핵심**: "위키는 지속적이고 복리적인 산물. LLM이 모든 것을 작성하고 유지."
- **3계층**: Raw Sources → Wiki → Schema
- **현재 구현**: Raw=memory/, Wiki=obsidian-vault/, Schema=TOOLS.md+MEMORY.md
- **평가**: Wiki/Schema 계층은 우수, Raw→Wiki 자동화 미흡

### gbrain (Garry Tan)
- **핵심**: 지식 그래프 + 하이브리드 검색 + 자동 엔티티 추출
- **주요 기능**:
  - Zero-LLM 엔티티 감지 + 타입화된 링크 (works_at, founded 등)
  - 하이브리드 검색 (pgvector + tsvector + RRF)
  - 26개 스킬 (ingest, enrich, maintain, query, publish 등)
  - 내결함성 스케줄러 (Minions 큐)
  - Compiled Truth + Timeline 분리
- **현재 상태**: PGLite WASM 오류로 전체 미도입, Flask+PostgreSQL로 우회

## 🎯 개선 필요 사항

### 단기 (이번 주)
1. NotebookLM을 `maintain` 스키처럼 활용 — 주간 모순/끊어진링크 탐지
2. NotebookLM을 지식 압축 엔진으로 활용 — 파편→Compiled Truth 변환
3. NotebookLM 업로드 자동화 파이프라인 구축

### 중기 (이번 달)
4. PostgreSQL에 pgvector 확장 → 하이브리드 검색 구현
5. 옵시디언 페이지에 Compiled Truth + Timeline 구조 도입
6. gbrain Docker 또는 Supabase로 재통합 시도

### 장기
7. gbrain 완전 통합 — 26개 스킬 활용
8. Zero-LLM 엔티티 자동 추출
9. Minions 큐 기반 내결함성 스케줄러

## 📓 NotebookLM의 역할
1. **Synthesis Engine**: Raw 데이터 → 정제된 마크다운 변환
2. **Health Checker**: 위키 모순/오래된 정보 탐지
3. **Query Engine**: 자연어로 전체 지식베이스 검색
4. **Report Generator**: 주간/월간 리포트 자동 생성
