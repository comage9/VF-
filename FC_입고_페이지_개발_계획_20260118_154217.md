## 🔧 구현 계획

### 1. 백엔드 API

#### a. Google Sheets 데이터 파싱
- CSV 다운로드 및 파싱
- SKU ID 기준으로 품목 분류 매핑

#### b. FC 입고 API 구현
- `/api/fc-inbound`
- `/api/fc-inbound/stats`
- `/api/fc-inbound/pivot`

#### c. 데이터베이스 스키마
```sql
-- 기존 BarcodeMaster 테이블 확장
CREATE TABLE IF NOT EXISTS "sales_api_barcodemaster" (
    "id" integer NOT NULL PRIMARY KEY AUTOINCREMENT,
    "sku_id" varchar(100) NOT NULL,
    "barcode" varchar(100) NOT NULL,
    "product_name" varchar(255) NOT NULL,
    "category" varchar(255) NOT NULL,  -- 대분류
    "subcategory" varchar(255) NOT NULL,  -- 중분류
    "color" varchar(100) NOT NULL,  -- 색상
    "location" varchar(255) NOT NULL
);

-- FC 입고 기록 테이블
CREATE TABLE IF NOT EXISTS "sales_api_fc_inbound" (
    "id" integer NOT NULL PRIMARY KEY AUTOINCREMENT,
    "sku_id" varchar(100) NOT NULL,
    "barcode" varchar(100) NOT NULL,
    "product_name" varchar(255) NOT NULL,
    "category" varchar(255) NOT NULL,
    "inbound_date" date NOT NULL,
    "quantity" integer NOT NULL,
    "logistics_center" varchar(100) NOT NULL
);
```
