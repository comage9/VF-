## 🔍 상세 분석

### 버그 1: 자동 검색 문제

#### 현재 코드 (라인 907-913)
```tsx
onChange={(e) => {
    const next = e.target.value;
    setSearchInput(next);
    if (next.length === 0 || next.length >= 2) {
        setSearchQuery(next);  // 자동으로 검색 실행
    }
}}
```

#### 문제점
- 사용자가 2글자 입력하면 즉시 `setSearchQuery(next)`가 호출되어 자동 검색 실행
- 엔터키 입력과 자동 검색이 중복되어 불필요한 API 호출 발생

#### 해결 방안
```tsx
onChange={(e) => {
    const next = e.target.value;
    setSearchInput(next);
    // 자동 검색 제거 - 엔터키로만 검색
}}
```

### 버그 2: 체크박스 필터링 문제

#### 현재 코드 (라인 1205-1219)
```tsx
{selectedCategories.length > 0 && canShowDailyPivot && (
    <div className="h-[500px] overflow-hidden">
        <CompactPivotTable
            title={`선택된 분류 품목별 일별 집계 (${selectedCategories.join(', ')})`}
            data={productPivot.filter((p: any) =>
                filtered.some(r => r.category === p.key && selectedCategories.includes(r.category))
            )}
            dates={dates}
            rowKey="key"
            rowLabel="품목"
            valueKey="quantity"
            secondaryValueKey="salesAmount"
        />
    </div>
)}
```

#### 문제점
1. **잘못된 필터링 로직:** `r.category === p.key` 비교는 카테고리명과 제품명을 비교하는 것으로 논리 오류
2. **데이터 구조 불일치:** 
   - `productPivot`: 제품별 일별 집계 데이터 (`key` = 제품명)
   - `filtered`: 필터링된 원본 레코드 (`r.category` = 카테고리명)
3. **결과:** 필터링 조건이 항상 실패하여 빈 배열이 전달됨 → 빈 화면

#### 데이터 구조 분석
```typescript
// productPivot 구조
productPivot = [
    {
        key: "제품A",           // 제품명
        values: {
            "2026-01-01": { quantity: 10, salesAmount: 100000 },
            "2026-01-02": { quantity: 15, salesAmount: 150000 },
            ...
        },
        total: { quantity: 25, salesAmount: 250000 }
    },
    ...
]

// filtered 구조
filtered = [
    {
        productName: "제품A",   // 제품명
        category: "카테고리1",   // 카테고리명
        quantity: 10,
        salesAmount: 100000,
        outboundDate: "2026-01-01"
    },
    ...
]
```

#### 해결 방안

**옵션 1: `filtered` 레코드에서 제품별 카테고리 매핑 생성**
```tsx
// 1. 제품별 카테고리 매핑 생성 (filtered 레코드에서)
const productCategoryMap = useMemo(() => {
    const map = new Map<string, string>();
    filtered.forEach(r => {
        if (r.productName && r.category) {
            map.set(r.productName, r.category);
        }
    });
    return map;
}, [filtered]);

// 2. 필터링 로직 수정
data={productPivot.filter((p: any) => {
    const productCategory = productCategoryMap.get(p.key);
    return productCategory && selectedCategories.includes(productCategory);
})}
```

**옵션 2: 선택된 카테고리의 제품들만 추출하여 새로운 피벗 생성**
```tsx
// 1. 선택된 카테고리에 속한 제품명들 추출
const selectedCategoryProducts = useMemo(() => {
    const products = new Set<string>();
    filtered.forEach(r => {
        if (selectedCategories.includes(r.category) && r.productName) {
            products.add(r.productName);
        }
    });
    return products;
}, [filtered, selectedCategories]);

// 2. 해당 제품들만 필터링
data={productPivot.filter((p: any) => selectedCategoryProducts.has(p.key))}
```

**권장: 옵션 1** (더 명확하고 유지보수에 용이)

## 📌 추가 고려사항

### 1. 빈 데이터 처리 (옵션)
선택된 카테고리에 해당하는 제품이 없는 경우, 빈 화면 대신 메시지 표시 고려:

```tsx
{selectedCategories.length > 0 && canShowDailyPivot && (
    <div className="h-[500px] overflow-hidden">
        {filteredProductPivot.length === 0 ? (
            <Card className="h-full flex items-center justify-center">
                <CardContent className="text-center text-gray-500">
                    선택된 분류에 해당하는 품목 데이터가 없습니다.
                </CardContent>
            </Card>
        ) : (
            <CompactPivotTable
                title={`선택된 분류 품목별 일별 집계 (${selectedCategories.join(', ')})`}
                data={filteredProductPivot}
                dates={dates}
                rowKey="key"
                rowLabel="품목"
                valueKey="quantity"
                secondaryValueKey="salesAmount"
            />
        )}
    </div>
)}
```

### 2. 디버깅 로그 추가 (개발 단계)
필터링 로직 디버깅을 위해 임시로 로그 추가 가능:

```tsx
// 개발 시 디버깅 용도
useEffect(() => {
    if (selectedCategories.length > 0) {
        console.log('선택된 카테고리:', selectedCategories);
        console.log('productPivot:', productPivot);
        console.log('productCategoryMap:', productCategoryMap);
        const filtered = productPivot.filter((p: any) => {
            const productCategory = productCategoryMap.get(p.key);
            return productCategory && selectedCategories.includes(productCategory);
        });
        console.log('필터링된 제품:', filtered);
    }
}, [selectedCategories, productPivot, productCategoryMap]);
```

### 3. 성능 최적화 고려
`productCategoryMap` 계산은 이미 `useMemo`로 최적화되어 있으므로 추가 최적화 불필요
