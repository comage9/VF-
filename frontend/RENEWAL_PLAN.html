4. 발견된 오류 · 문제점 상세
치명적 오류 (Critical — 즉시 수정 필요)
CRITICAL 중복 라우트 정의 → Silent Failure
CLAUDE.md에 명시: frontend/server/routes.ts에 중복 라우트 정의가 있으며, 이로 인해 일부 API 요청이 응답 없이 실패함. 특히 /api/outbound 계열에서 발생 가능.
📄 frontend/server/routes.ts — 전체 라우트 중복 검사 필요
CRITICAL RangeError: Invalid time value (날짜 처리 오류)
OutboundDashboardUnified.tsx의 eachDayOfInterval 함수에 null/undefined 날짜가 전달될 때 프론트엔드 전체가 크래시됨. isValid() 체크가 모든 날짜 파이프라인에 적용되지 않은 지점이 존재.
📄 frontend/client/src/components/outbound-dashboard-unified.tsx
CRITICAL DEBUG=True 프로덕션 노출
Django 설정이 DEBUG=True로 고정되어 있으며, 공개 서버(bonohouse.p-e.kr)에 그대로 배포됨. SQL 에러, 스택 트레이스, 내부 경로가 외부에 노출될 위험.
📄 backend/config/settings.py
CRITICAL CORS 전체 허용
CLAUDE.md "Security Notes": "CORS enabled for cross-origin requests (review for production)" — 현재 django-cors-headers가 모든 오리진을 허용하는 상태로 추정. 외부 공격에 취약.
📄 backend/config/settings.py — CORS_ALLOW_ALL_ORIGINS 확인 필요
CRITICAL .env 토큰 Git 노출 위험
CLAUDE.md에 "GitHub token is stored in .env as gittoken" 명시. .gitignore 처리 여부 및 실수로 커밋된 이력 확인 필요. Google Sheets API Key도 동일.
📄 .env, .gitignore
CRITICAL 생산계획 페이지 완전 미연동
현재 bonohouse.p-e.kr:5174의 생산계획 페이지가 Django API(/api/production/*)와 연동되지 않고 별도 PMS 시스템(정적 HTML+RESTful Table API)으로 동작. 데이터 이중화 및 불일치 발생.
📄 production.html (PMS) vs Django ProductionLog 모델
CRITICAL 인증 시스템 부재
Django 및 Node.js 서버 모두 인증/세션 없이 모든 API가 공개 상태. 생산계획 수정·삭제 등 민감 작업이 무제한 허용됨.
📄 backend/sales_api/views.py, frontend/server/routes.ts
CRITICAL 대용량 데이터 프론트 처리
200K+ 레코드를 일부 컴포넌트(CompactPivotTable)에서 프론트엔드로 전량 수신 후 처리. 60일 이상 범위에서 브라우저 메모리 한계 도달. 임시로 60일 이상 시 숨김 처리만 적용된 상태.
📄 frontend/client/src/components/ — CompactPivotTable 관련 전체
고우선 개선사항 (High)
HIGH UI 디자인 시스템 불일치 (2개 시스템 혼재)
기존 VF 대시보드(Tailwind CSS + shadcn/ui + Recharts)와 PMS(Noto Sans KR + 커스텀 CSS + Chart.js)가 완전히 다른 디자인 언어를 사용. 사용자가 동일 도메인에서 완전히 다른 UI를 경험하게 됨.

  • 색상: VF(회색 계열 다크) vs PMS(블루 라이트)
  • 폰트: system-font vs Noto Sans KR
  • 차트: Recharts vs Chart.js
