# 프로젝트 설명서 (리뉴얼 버전)

## 1. 프로젝트 개요

이 프로젝트는 `VF`라는 회사의 판매 데이터를 시각화하고 분석하기 위한 대시보드 애플리케이션입니다. 초기에는 Node.js 기반의 간단한 백엔드와 React 기반의 프론트엔드로 구성되었으나, 데이터 양이 증가하고 성능 문제가 발생함에 따라 Django 기반의 고성능 백엔드로 전환하고 프론트엔드 최적화를 진행하였습니다.

**주요 목적:**
*   **판매 데이터 시각화:** 일별, 주별, 월별 판매량을 그래프로 직관적으로 표시.
*   **재고 관리 효율화:** 재고 현황 및 출고 데이터를 효율적으로 관리하고 분석.
*   **성능 최적화:** 대량의 데이터를 안정적으로 처리하고 빠른 응답 속도를 제공.
*   **사용자 경험 개선:** 데이터 조회 범위 확장에 따른 `RangeError` 해결 및 UI/UX 개선.

## 2. 아키텍처 (리뉴얼 후)

하이브리드 아키텍처를 채택하여 Node.js와 Django를 함께 사용합니다.

*   **프론트엔드 (Frontend):**
    *   **기술 스택:** React (Vite를 사용하여 빌드).
    *   **역할:** 사용자 인터페이스 렌더링, 사용자 입력 처리, 백엔드 API와의 통신.
    *   **주요 최적화:**
        *   대량 데이터셋에 대한 차트 다운샘플링 (점/애니메이션 제거, 라벨 수 제한).
        *   60일 이상 데이터 조회 시 `CompactPivotTable` 컴포넌트 숨김 처리.
        *   날짜 유효성 검사 (`isValid` 체크) 추가를 통해 `RangeError: Invalid time value` 오류 해결.

*   **백엔드 (Backend):**
    *   **기술 스택:** Django 6.0, Django REST Framework, SQLite (데이터베이스).
    *   **역할:**
        *   데이터 저장 및 관리 (SQLite).
        *   프론트엔드 요청에 대한 데이터 처리 및 제공.
        *   대용량 데이터에 대한 서버 사이드 집계 (일별/주별/월별 그룹화)를 통해 데이터 처리 속도 향상.
        *   `/api/outbound/stats`와 같은 고성능 집계 API 구현.
    *   **데이터 이관:** 기존 JSON/SQLite 형태의 레거시 데이터를 Django 모델 기반의 SQLite DB로 마이그레이션 완료 (20만 건 이상의 레코드 처리).

*   **Node.js 서버:**
    *   **역할:** 정적 파일(프론트엔드 빌드 결과물) 제공, Django 백엔드 API로의 프록시 역할.
    *   **설정:** `server.js` 파일에서 `/api/outbound` 및 `/api/outbound/stats` 경로를 `http://localhost:5176` (Django 백엔드)로 프록시하도록 설정.

## 3. 리뉴얼 코드의 특징 및 개선점

### 3.1. 안정성 및 오류 처리

*   **`RangeError: Invalid time value` 해결:** `OutboundDashboardUnified.tsx` 파일에 `isValid` 검사 로직을 추가하여 `eachDayOfInterval` 함수에 유효하지 않은 날짜 값이 전달되는 것을 방지했습니다. 이는 데이터 로딩 중이나 데이터가 없는 경우에 발생할 수 있는 프론트엔드 크래시를 방지합니다.

### 3.2. 성능 최적화

*   **백엔드 데이터 집계:** 프론트엔드에서 처리하던 복잡한 데이터 집계 로직을 Django 백엔드로 이동시켜 서버 자원을 효율적으로 사용하고, 프론트엔드의 부하를 줄였습니다.
*   **프론트엔드 렌더링 최적화:**
    *   차트 시각화 시 데이터 포인트 수를 줄이는 다운샘플링 기법을 적용했습니다.
    *   불필요한 애니메이션이나 과도한 라벨링을 제거하여 렌더링 속도를 높였습니다.
    *   넓은 범위의 데이터를 조회할 때 성능에 부담을 주는 `CompactPivotTable` 컴포넌트를 자동으로 숨기도록 하여, 사용자가 필요한 정보에 집중할 수 있도록 하면서도 성능 저하를 방지했습니다.

### 3.3. 확장성 및 유지보수성

*   **Django REST Framework 활용:** RESTful API를 표준화하여 백엔드와 프론트엔드 간의 통신을 명확히 하고, 향후 기능 확장 및 유지보수를 용이하게 했습니다.
*   **모듈화된 코드베이스:** 백엔드와 프론트엔드가 명확히 분리되어 각자의 책임 영역을 가지므로, 개별 모듈의 개발 및 테스트가 독립적으로 가능합니다.

## 4. 실행 방법

### 백엔드 (Django)

1.  `renewal/backend` 폴더로 이동합니다.
2.  가상 환경을 활성화합니다.
    ```bash
    .\venv\Scripts\activate
    ```
3.  필요한 Python 패키지를 설치합니다.
    ```bash
    pip install -r requirements.txt
    ```
4.  데이터베이스 마이그레이션을 실행합니다.
    ```bash
    python manage.py migrate
    ```
5.  Django 서버를 실행합니다.
    ```bash
    python manage.py runserver 0.0.0.0:5176
    ```
    (데이터 마이그레이션이 필요한 경우 `python manage.py import_legacy_db`를 실행하여 기존 데이터를 가져올 수 있습니다.)

### 프론트엔드 (React)

1.  `renewal/frontend` 폴더로 이동합니다.
2.  의존성 패키지를 설치합니다.
    ```bash
    npm install
    ```
3.  프로덕션 빌드를 생성합니다.
    ```bash
    npm run build
    ```
    (개발 환경에서 실행하려면 `npm run dev`를 사용하고, Node.js 서버 없이 직접 Vite 개발 서버를 실행할 수 있습니다.)

### Node.js 프록시 서버

1.  프로젝트 루트 디렉토리로 이동합니다.
2.  `server.js` 파일을 실행합니다.
    ```bash
    node server.js
    ```
    이 서버는 프론트엔드 빌드 결과물을 서빙하고, `/api/outbound` 경로로 들어오는 요청을 Django 백엔드로 포워딩합니다.

## 5. 결론

리뉴얼된 프로젝트는 성능, 안정성 및 유지보수성을 크게 향상시켰습니다. 특히 대용량 데이터 처리와 `RangeError`와 같은 고질적인 문제들을 해결하여 더욱 견고한 애플리케이션으로 거듭났습니다.
